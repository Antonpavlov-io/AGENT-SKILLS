# Скилл: Управление ветками

## Типы веток и соглашения об именовании

| Тип | Префикс | Назначение |
|-----|---------|------------|
| Главная | `main` | Стабильная версия; коммиты только через PR |
| Функция | `feature/<name>` | Разработка новой функциональности |
| Баг | `fix/<name>` | Исправление ошибок |
| Критический баг | `hotfix/<name>` | Срочное исправление в продакшне |
| Релиз | `release/<version>` | Финальное тестирование перед выпуском |
| Документация | `docs/<name>` | Только изменения документации |
| Эксперимент | `experiment/<name>` | R&D; слияние в `main` не гарантируется |

## Правила

1. Прямые коммиты в `main` запрещены — только через pull request.
2. Имя ветки: строчные буквы, слова через дефис, с префиксом из таблицы выше.
3. Одна ветка — одна задача; не смешивать feature и fix.
4. Ветку регулярно синхронизировать с `main` командой `git rebase main`, чтобы избежать конфликтов при слиянии.
5. После успешного слияния PR ветку удалять.
6. Включить branch protection rules для `main` в настройках репозитория.
