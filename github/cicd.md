# Скилл: CI/CD

## Что это

CI/CD — автоматизированный конвейер сборки, тестирования и развёртывания.  
Инструмент: **GitHub Actions** (файлы в `.github/workflows/`).

## CI — Continuous Integration

Запускается при каждом коммите или PR в `main`.

Шаги:
1. Запуск автотестов.
2. Статический анализ кода (линтинг, поиск уязвимостей).
3. Сборка образа / артефакта приложения.
4. Если любой шаг упал — разработчик получает уведомление, слияние блокируется.

## CD — Continuous Deployment

Запускается после успешного CI в `main`.

Шаги:
1. Деплой собранного артефакта на staging / production.
2. Проверка работоспособности (health check).
3. При неудаче — автоматический откат к предыдущей версии.

## Хранение секретов

Токены, пароли, API-ключи хранить **только** в:  
`Settings → Secrets and variables → Actions` репозитория.  
Никогда не коммитить в код.

## Правила для агента

- Не изменять файлы в `.github/workflows/` без явного указания.
- При добавлении новых секретов документировать их в README или в описании PR.
- Перед пушем в `main` убедиться, что тесты проходят локально.
