# Скилл: Коммиты

## Формат сообщения

```
<prefix>: <краткое описание на русском>
```

Примеры:
- `feat: добавлена авторизация через OAuth`
- `fix: исправлена ошибка сортировки в модуле отчётов`
- `docs: обновлена документация API`
- `refactor: упрощена логика обработки запросов`
- `test: добавлены тесты для модуля платежей`
- `chore: обновлены зависимости`

## Префиксы

| Префикс | Когда использовать |
|---------|--------------------|
| `feat` | Новая функциональность |
| `fix` | Исправление бага |
| `docs` | Только документация |
| `refactor` | Рефакторинг без изменения поведения |
| `test` | Добавление или изменение тестов |
| `chore` | Настройки, зависимости, CI |

## Правила

1. Язык коммитов — **русский**.
2. Один коммит — одно логическое изменение. Не смешивать разные задачи.
3. Если затронуты несколько логических частей проекта (например, контроллеры и модели) — делать **отдельный коммит на каждую часть**.
4. Коммит делать после завершения каждой итерации разработки по спецификации.
5. При работе в feature-ветке — коммитить в неё; в `main` только через PR.

## Категорически запрещено включать в коммит

- Переменные окружения из `.env` и любые их значения.
- Пароли, токены, API-ключи, секреты.
- Файлы с credentials (JSON с ключами сервисных аккаунтов и т.п.).

Все такие паттерны добавить в `.gitignore` до первого коммита.
